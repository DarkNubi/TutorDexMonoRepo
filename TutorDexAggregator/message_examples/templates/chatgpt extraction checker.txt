## KEY EXTRACTION RULES

### 1. Atomic extraction

Each field stands alone. Never infer from others.

### 2. Lists

* Use **null**, not `[]`, for missing optional lists
  (level, specific_student_level, student_gender, address, postal_code, nearest_mrt, tutor_type, slot_details)
* `subjects` is required → use `[]` if none confidently extracted
* Deduplicate case-insensitively, preserve first casing and order

### 3. Booleans

* `is_tuition_centre = true` **only if explicitly stated**, else false

### 4. Canonical values only

* student_gender: Male / Female
* tutor_gender: Male / Female (any / no preference → null)
* learning_mode.mode: Online / Face-to-Face / Hybrid
* rate.rate_type: hourly / per_session / unknown

### 5. Academic info

* Preserve all wording in academic_display_text
* Do NOT force interpretations (G3, NA, NT, IP, streams)
* Extract structured signals only if explicit and unambiguous

### 6. Subjects & levels

* NON-AUTHORITATIVE tags only
* Include only if explicit or unambiguous
* If unsure → omit subject, set level fields to null

### 7. Learning mode

* raw_text copied verbatim if present
* mode only if explicit; else null

### 8. Lesson schedule

* raw_text copied verbatim if present
* Extract numbers only when quantity + unit are clear
* total_hours_per_week only if lessons_per_week AND hours_per_lesson are explicit and related
* per_subject true/false only if explicitly stated
* subject_breakdown only if explicit

### 9. Time availability (fixed shape)

* explicit & estimated MUST exist
* 7 days, each always an array
* Slot format: HH:MM-HH:MM
* Normalize day names
* daily/everyday → all 7 days
* If no usable timing → arrays empty, note captures wording

### 10. Explicit timing

* Only concrete day + time ranges
* Single time + duration → compute end
* Single time w/o duration → start = end, preserve note
* Do NOT move explicit → estimated
* Group terms (e.g. weekdays 7pm) → estimated

### 11. Estimated timing

* Only fuzzy terms (morning, evening, after/before, weekdays/weekends)
* Never derived from explicit timing
* Rules:

  * after HH:MM → HH:MM–23:00
  * before HH:MM → 08:00–HH:MM
  * morning 08–12, afternoon 12–17, evening 16–21, night 19–23
  * weekdays Mon–Fri 08–23, weekends Sat–Sun 08–23
  * after HH:MM w/o end → HH:MM-23:00; before HH:MM w/o start → 08:00-HH:MM.

### 12. Time format

* 24-hour HH:MM, leading zeros
* Convert am/pm
* If unclear → do not guess, use note

### 13. Rates

* raw_text preserved verbatim
* min = lowest explicit, max = highest explicit (any tier)
* One rate → min = max
* None → min/max null
* rate_type only if explicit; else unknown

### 14. Address / MRT / postal

* address: venues/areas/streets as stated
* nearest_mrt ONLY if MRT explicitly mentioned
* postal_code ONLY explicit 6-digit strings

### 15. Start date

* YYYY-MM-DD only if full date explicit
* Otherwise preserve wording (ASAP, Jan 2026)

### 16. Slot details

* Strict & optional
* Output only if ALL fields explicit per slot
* Else null

### 17. Facts only

No UI text or interpretation.

### 18. Output rules

* Exactly ONE JSON object
* No comments, markdown, or extra keys
* Exact key order
* No trailing commas
* Stop after final `}`
